Request Data Flow

StepFromToWhat happens
1 User types message Browser → Flask → POST /chat with JSON body {message: "..."}
2 Flask Crisis Detector → Scans message against 40+ keywords → returns SAFE / DISTRESS / CRISIS
3 Crisis Detector Data Layer → If CRISIS: logs event to crisis_log.json with timestamp + keywords
4 Flask MentalHealthAgent → Agent builds dynamic system prompt based on risk level
5 Agent IBM IAM → Exchanges API key for Bearer access token
6 WatsonX Client IBM WatsonX AI → Sends full conversation history + system prompt → receives AI reply
7 WatsonX AI Agent → If CRISIS: appends mandatory crisis hotlines footer to reply
8 Flask Browser → Returns {reply, risk_level, keywords} — UI renders chat bubble

Core Modules

app.py
Flask web server. All REST API routes. Error handling for rate limits and connection failures.
mental_health_agent.py
Core agent loop. Manages conversation history, builds system prompts, tracks peak risk level per session.
watsonx_client.py
IBM WatsonX API client. IAM token exchange, chat completions, retry logic with exponential backoff on 429.
crisis_detector.py
Keyword scanner. Returns SAFE, DISTRESS, or CRISIS risk level + matched keyword list from every message.
mood_tracker.py
Reads and writes mood_history.json, journal.json, crisis_log.json. Generates 7-day summary reports.
resources.py
Static data: crisis hotlines for 7 countries, 10 wellness tips, 3 grounding exercises (box breathing, 5-4-3-2-1, safe place).

Risk Detection Engine

LevelTriggered bySystem PromptResponse Behaviour
SAFE No risk keywords matched Base empathy prompt Warm, supportive conversation. Temp: 0.75
DISTRESS hopeless, anxious, lonely, trapped, overwhelmed… Base + distress guidance Validation + grounding offer + professional help reminder. Temp: 0.75
CRISIS suicide, end my life, kill myself, no reason to live… Base + crisis protocol Crisis empathy + mandatory hotlines footer appended. Temp: 0.65. Event logged.
